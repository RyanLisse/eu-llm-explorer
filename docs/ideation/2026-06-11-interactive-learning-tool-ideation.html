Interactive Learning Tool — EU LLM Explorer

date 2026-06-11 topic interactive-learning-tool mode repo-grounded

Focus: make this an interactive learning tool, with light/dark mode and easy UX and UI, remove all redundant files, and create an amazing experience.

Grounding Context

Codebase Context. Next.js (App Router) + Effect-TS + Turso/libSQL comparison tool for EU-sovereign LLM routes. Split-pane UI: app/Explorer.tsx (56 KB monolith) with tabs/filters on the left, agent-native chat panel (Chat.tsx, AI SDK v5 with a set_filters tool mirroring FilterState in src/atoms.ts) on the right. Dark-only Blinqx/HippoLine theme: all colors are CSS variables under :root in app/globals.css (56 KB) — token indirection exists but no light token set. Tier semantics are color-encoded (Tier A growth green, Tier B sky, Tier C coral). Conventions to preserve: page.tsx is a pure server data loader, PageShell.tsx owns client layout state, module-level keep-alive atoms, no TanStack Query. Hygiene findings: four .tmp-*.png screenshots (~1.2 MB) at repo root, duplicate "(1)" files in docs/research/, and ResearchBriefing.tsx overlapping the chat/briefing content.

Past learnings. No docs/solutions/ exists. docs/design.md records the dark palette as a deliberate brand decision (aligned to eval-bench's official Blinqx palette, commit b21a323); grain overlay and glassmorphism are tuned for dark surfaces. The chat plan doc records that any filter-field rename breaks the chat tool contract.

External context. The Pudding's story structures (stacked single-chart, scrollytelling, step-progression; "sledgehammer stat" first). NYT finding: ~85% of readers ignore tooltips — surface insights inline, never gate them behind interaction. Our World in Data's declarative chart titles (full-sentence conclusions). Distill.pub's reactive-diagram pattern. Khan/Brilliant evidence: prediction prompts beat scored quizzes (quizzes cause drop-off). next-themes is the standard no-flash dark-mode pattern; Driver.js the permissive-license tour library; "empty state as first lesson" is the 2025 consensus for data tools.

1.

Learn Mode: a chaptered data story that drives the real explorer

axis: guided learning layer confidence 75% complexity High
Description
Add a "Learn" mode alongside the existing explorer: a sequence of named chapters ("Why EU sovereignty matters", "The three tiers", "Reliability vs. residency", "Pick your route"), each pairing 2–3 sentences of narrative with a live explorer state. Advancing a chapter sets the real filters via the existing FilterState atom — the story and the tool are the same surface, not a separate onboarding deck. Chapters are defined as data (a JSON/TS array of {title, prose, filterState}), so new lessons are content, not code. A chapter sidebar shows position and progress.
Basis
external: The Pudding's step-based progression and Distill.pub's reactive-document pattern; Observable's named-chapter navigation. direct: the keep-alive filterAtom and the chat panel's set_filters tool prove the app already supports programmatic filter state — the story engine reuses the exact mechanism the agent uses.
Rationale
This is the single move that converts a dashboard into a learning tool rather than decorating it with tips. Because chapters are data driving existing state, it compounds: every future insight becomes a one-object lesson, and the chat agent can reuse chapters as tours.
Downsides
Largest build on the list; needs editorial writing, not just code. Chapter definitions must stay in sync with FilterState shape (same coupling the chat tool already has).
2.

Light/dark theme on the existing token layer

axis: theming confidence 90% complexity Medium
Description
Add a light token set under [data-theme="light"] in app/globals.css, a toggle in PageShell.tsx (the documented client-state owner), system-preference default, and a blocking inline script in the root layout to prevent flash-of-wrong-theme (the next-themes pattern, with or without the library). Preserve Tier A/B/C semantic colors in both themes, sourcing light values from the Blinqx palette. Audit dark-coupled effects — grain overlay, glassmorphism, glow — and give each a light variant or remove it in light mode.
Basis
direct: every color already routes through :root CSS variables, so a second token set is the designed-for extension point; docs/design.md confirms the dark palette is deliberate brand, not accident. external: next-themes no-flash pattern is the 2025–26 standard.
Rationale
Explicitly requested, high-confidence, and the token indirection means the cost is concentrated in one file plus a small toggle — no component restyling. A light theme is also table stakes for a learning tool used in daylight classrooms/offices.
Downsides
Choosing a light palette that keeps tier-color contrast accessible takes real design care; the grain/glass effects need per-theme judgment calls.
3.

Insight-first UX: declarative titles, takeaway cards, plain-language summaries

axis: explorer UX confidence 85% complexity Medium
Description
Three coordinated changes: (a) every chart/table heading becomes a full-sentence conclusion that updates with the data; (b) a live plain-language summary line of the current filter result ("3 EU-sovereign routes match · best reliability 94/100"); (c) takeaway cards after key views — one-sentence conclusion plus supporting stat, with the filter state encoded in the URL so any takeaway is linkable and shareable.
Basis
external: Our World in Data's declarative-title practice; NYT's 85%-ignore-tooltips finding argues for inline text insights over hover reveals; museum interpretive-label layering (caption → paragraph → deep dive). direct: the recommendation/reliability services already compute the conclusions — they're just not verbalized in the UI.
Rationale
Cheapest unit of "learning tool" value: non-experts get the conclusion without learning the interface first, experts lose nothing. Pairs multiplicatively with Learn Mode (chapters reuse takeaway cards) and the chat agent (same sentences become its narration).
Downsides
Generated sentences must stay honest as filters change — template logic needs care at edge cases (zero results, single result).

4.

Chat as tour guide: starter prompts and agent-driven walkthroughs

axis: chat as teacher confidence 75% complexity Medium
Description
Lean into the existing agent-native chat as the teaching mechanism: curated starter prompts ("Show me the safest EU-only route for coding", "Why is Tier C excluded?") rendered as one-click chips in the chat panel; the agent answers and drives the filters via set_filters, narrating what it changed. Add a "walk me through it" prompt that steps through the same chapter definitions as Learn Mode.
Basis
direct: the chat plan doc already establishes per-send filter-context injection and the set_filters deep-merge tool — the brainstorm doc itself notes "the agent narrating and driving filters is itself a teaching mechanism." reasoned: watching the UI respond to a question teaches the interface and the data simultaneously, which static tours cannot.
Rationale
Highest-leverage reuse in the repo: the hard infrastructure (tool contract, SQL guard, context injection) is built; what's missing is curated entry points that make a first-time user actually experience it.
Downsides
LLM answer latency/cost per walkthrough; prompts need curation so the agent's filter moves stay predictable.
5.

First-run experience: curated default view plus dismissible micro-tour

axis: explorer UX confidence 80% complexity Low
Description
Treat the first load as the first lesson: open on a pre-filtered, opinionated view (e.g., "EU-sovereign routes ranked by reliability") with a one-line callout ("This is the Tier-A shortlist — try widening to Tier B"), plus an optional 3–4 step Driver.js tour anchoring on the filter sidebar, the scatter/table, and the chat panel. Dismiss persists; never blocks.
Basis
external: "empty state as first lesson" consensus (Eleken/Raw Studio); Driver.js as the permissive-license tour choice. reasoned: the app currently opens as a filter wall — the highest-bounce moment for non-experts is the first ten seconds.
Rationale
Smallest change with the largest effect on "easy UX" for new users; also the natural front door into Learn Mode and the chat starter prompts.
Downsides
Tour anchors couple to DOM structure — keep steps few and resilient.
6.

Prediction prompts: guess before you see

axis: guided learning layer confidence 70% complexity Low–Medium
Description
At chapter transitions (and optionally before headline charts), pose a single-click prediction: "Before we look — do you think EU-sovereign routes are more or less reliable than US hyperscaler routes?" Lock the pick, reveal the actual data, no right/wrong scoring — just the reveal and "X% guessed this." A small React state machine, no library.
Basis
external: retrieval-practice/testing-effect literature via Khan Academy and Brilliant: prediction prompts drive engagement where scored multiple-choice causes drop-off; single-click input avoids the friction failure mode.
Rationale
The one mechanic that makes reading active — it converts "scrolling past a chart" into a small commitment the reveal then pays off. Distinct from, and composable with, Learn Mode chapters.
Downsides
Needs restraint — more than a handful per session becomes a gimmick; "X% guessed" requires either storage or honest omission at first.
7.

Repo cleanup and decomposition

axis: codebase hygiene confidence 95% complexity Low–Medium
Description
Delete the four root-level .tmp-*.png screenshots (~1.2 MB); consolidate the duplicate "(1)" files in docs/research/; fold ResearchBriefing.tsx into the Learn Mode/briefing flow or remove it; split Explorer.tsx (56 KB) into scatter, table, filter-sidebar, and tab components; extract design tokens from the 56 KB globals.css into a dedicated token block/file. Preserve the documented conventions (server loader, client shell, keep-alive atoms).
Basis
direct: all six findings come from the codebase scan — file sizes, the tmp screenshots, and the duplicate docs are observable in the tree today.
Rationale
Explicitly requested ("remove all redundant files"), and the decomposition is the enabler for everything above: Learn Mode, theming, and insight cards all touch Explorer.tsx — splitting it first makes every other idea cheaper and safer. Tactical scope acknowledged; meeting-test waived per the stated focus.
Downsides
Decomposition of a 56 KB component risks regressions without visual checks; do it before, not during, feature work.
