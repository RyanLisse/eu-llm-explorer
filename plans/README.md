## Findings considered and rejected

- **Direct content-tab empty data bug**: rejected after browser smoke. Starting at `/?tab=presentation` and clicking Compare rendered vendor data correctly.
- **No auth on `/api/chat` as a standalone finding**: rejected for this plan set because the original chat requirements explicitly marked authentication out of scope. Security work here is scoped to validation, prompt/data isolation, and SQL read-surface restriction.
- **Local `.env.local` credentials as committed-secret finding**: rejected as a code finding because `.env.local` is ignored. Operators should still rotate the listed credential types if this worktree crossed a trust boundary.
- **PDF decision-packet export**: rejected for first implementation. Markdown/JSON is lower-risk, auditable, testable, and enough for governance handoff.
- **Fixing the moderate Next/PostCSS advisory inside Plan 001**: rejected unless a safe Next patch is available. Plan 001 may document the audit baseline; dependency remediation can be separate if the advisory persists.
- **Treating Compare/Advanced filter separation as intentional forever**: rejected after the 2026-06-19 audit. `docs/design.md` presents both as parallel workflows, while chat currently edits only Advanced filter state; this is real product friction now that Compare is the default tab.
- **Keeping `auditOfficialVendorPresence()` as a harmless placeholder**: rejected after the 2026-06-19 audit. A check that always audits an empty platform set is misleading in a repository whose value proposition depends on source freshness.
